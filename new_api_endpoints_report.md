# تقرير الواجهات البرمجية الجديدة (APIs) لخدمة شحن المنتجات وتأكيد الـ OTP

تم بنجاح بناء وتهيئة المسارات والواجهات البرمجية الجديدة في الباك اند لدعم ميزة **شحن المنتجات** ودورة التحقق الثنائي عبر أكواد الـ **OTP** (الرمز السري المؤقت). 

يوضح هذا التقرير تفاصيل هذه الواجهات البرمجية، وظيفة كل منها، وموقعها داخل كوليكشن **Bruno** لتسهيل ربطها مع تطبيق الفلاتر (Flutter).

---

## تفاصيل الواجهات البرمجية (Detailed API Specifications)

### 1. واجهة رفع الهوية الوطنية (`update-national-id`)
* **الهدف**: توثيق حساب العميل (الراسل) برفع الرقم القومي وصور بطاقة الهوية لمرة واحدة لضمان قانونية الشحنات قبل تمكينه من إنشاء طلبات شحن المنتجات.
* **نوع الطلب**: `POST` (صيغة `multipart-form`)
* **المسار**: `{{url}}/user/profile/update`
* **المعاملات المطلوبة (Body Parameters)**:
  * `national_id` (نص): الرقم القومي الخاص بالمستخدم.
  * `national_id_front` (ملف/صورة): صورة وجه بطاقة الهوية الشخصية.
  * `national_id_back` (ملف/صورة): صورة ظهر بطاقة الهوية الشخصية.
  * `national_id_selfie` (ملف/صورة): صورة سيلفي شخصية للمستخدم وهو يحمل بطاقته بجانب وجهه.
* **الرد المتوقع (Response)**: بيانات المستخدم كاملة من جدول `users` متضمنة الروابط المباشرة للصور المرفوعة.

### 2. واجهة تأكيد وصول السائق للراسل (`driver arrive sender`)
* **الهدف**: يستخدمها السائق عند وصوله الفعلي لموقع العميل الراسل لتنبيهه بأن السائق بانتظار استلام الشحنة منه.
* **نوع الطلب**: `POST`
* **المسار**: `{{url}}/order/{order_id}/driver-arrive-sender`
* **التأثير في الباك اند**:
  * تحديث توقيت الوصول `driver_arrived_at_sender_at` إلى الوقت الحالي.
  * تحديث حالة الطلب إلى `arrived`.
  * بث التحديث بالريل تايم (Pusher) وإرسال إشعار فوري (Push Notification) للراسل.

### 3. واجهة التحقق من كود الاستلام وبدء الرحلة (`verify pickup otp`)
* **الهدف**: تأكيد السائق أنه استلم الشحنة من الراسل فعلياً بعد كتابة الـ OTP المكون من 4 أرقام والذي يمليه عليه الراسل.
* **نوع الطلب**: `POST` (JSON)
* **المسار**: `{{url}}/order/{order_id}/verify-pickup-otp`
* **محتوى الطلب (Body)**:
  ```json
  {
    "pickup_otp": "1234"
  }
  ```
* **التأثير في الباك اند**:
  * مطابقة الكود المدخل مع كود الاستلام المخزن بالرحلة.
  * تحديث تواقيت الاستلام: `sender_confirmed_handover_at` و `driver_confirmed_pickup_at`.
  * **إذا كان الدفع نقداً (Cash)**: يتم تأكيد استلام السائق للمبلغ تلقائياً وتحديث حقل التوقيت `driver_confirmed_cash_at`.
  * تحويل حالة الرحلة إلى `on_trip` وبدء التوصيل رسمياً.

### 4. واجهة تأكيد وصول السائق للمستلم (`driver arrive receiver`)
* **الهدف**: يستخدمها السائق عند وصوله لموقع المستلم النهائي لتحديث خريطة المستلم والراسل وإبلاغه بوصول الشحنة.
* **نوع الطلب**: `POST`
* **المسار**: `{{url}}/order/{order_id}/driver-arrive-receiver`
* **التأثير في الباك اند**:
  * تحديث توقيت وصول المستلم `driver_arrived_at_receiver_at`.
  * إرسال إشعارات فورية لكل من الراسل والمستلم (إذا كان مسجلاً برقم هاتفه بالتطبيق).

### 5. واجهة التحقق من كود التسليم وإنهاء الرحلة (`verify delivery otp`)
* **الهدف**: تأكيد السائق تسليم الطرد للمستلم بنجاح بعد كتابة كود التسليم النهائي (`delivery_otp`) المأخوذ من المستلم.
* **نوع الطلب**: `POST` (JSON)
* **المسار**: `{{url}}/order/{order_id}/verify-delivery-otp`
* **محتوى الطلب (Body)**:
  ```json
  {
    "delivery_otp": "5678"
  }
  ```
* **التأثير في الباك اند**:
  * مطابقة الكود وتحديث التواقيت: `driver_confirmed_delivery_at` و `receiver_confirmed_delivery_at` وتوقيت إنهاء الرحلة.
  * تحديث حالة الرحلة إلى `completed`.
  * **الحسابات المالية**: تحصيل عمولة النظام وتوزيع المبالغ المالية (إطلاق رصيد الضمان Escrow والـ Payouts لمحفظة السائق).
  * توزيع نقاط التقييم والمكافآت (Gamification) لكلا الطرفين.

## إرشادات هامة لمطور الفلاتر (Flutter Integration Tips)

1. **حماية أكواد الـ OTP**:
   * تم تطبيق حماية صارمة في مورد البيانات `OrderResource` بالباك اند؛ حيث لا يتم إرسال كود `pickup_otp` ولا كود `delivery_otp` في مصفوفة بيانات الرحلة المرسلة للسائق.
   * يتم إرسالها فقط للعميل صاحب الشحنة (الراسل) أو المستلم (المرسل إليه).
   * بالتالي، يتوجب على مطور التطبيق عمل واجهة للسائق يطلب فيها إدخال الكود شفهياً من العميل لتمريره للـ API.
2. **رابط تتبع المرسل إليه**:
   * عند إسناد الرحلة للسائق، يتم إرسال رسالة SMS للمستلم تحتوي على الرابط:
     `https://shakshak.net/track/{order_id}`
   * يجب تهيئة الـ **Deep Linking** في تطبيق الفلاتر ليلتقط هذا الرابط ويقوم بتوجيه المرسل إليه (بعد تسجيل دخوله التقليدي بالـ OTP برقم هاتفه) مباشرة إلى شاشة التتبع المباشر الخاصة به.
